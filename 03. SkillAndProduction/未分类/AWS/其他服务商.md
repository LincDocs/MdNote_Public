# 其他服务商

## 服务商 - 两种备案问题

在开始操作前，需要先确定 阿里云/腾讯云/AWS/cloudflare 等是否支持你想要的域名 (大家支持的都不同)

需要注意的是：有理解两个概念:

| 概念    | **域名实名认证** (Domain Verification)        | **网站ICP备案** (ICP Filing)   |
| ----- | --------------------------------------- | -------------------------- |
| 针对什么？ | **域名本身** (你的“门牌号”)                      | **网站服务器** (你的“房子”)         |
| 谁来要求？ | **中国的域名注册管理机构**<br>(简称**注册局**)          | **中国的工信部**                 |
| 触发条件？ | 只要你的域名是**在中国大陆服务商**（如阿里云、腾讯云）**注册购买的**。 | 只要你的网站**服务器是放在中国大陆境内**的。   |
| 简单比喻  | 好比在中国买车，必须上一个“车牌实名登记”。                  | 好比在中国大陆的公路上开车，必须要有“行车许可证”。 |

国内购买域名与购买服务器都需要实名。由于注册局规定，国内注册域名后，未实名时域名会被锁定。
包括阿里云、腾讯云、国内版的AWS

注意如果国内购买域名后，还会由于注册局的限制，要**60天后**才能移出

## 服务商 - 支持的域名问题

有的域名他们可能都没有，如 obsidian.md 中的 `.md` 域名，需要到 https://nic.md/en/login/ 进行注册 (仅授权给了部分机构)

其实 `.md` 是摩尔多瓦（Moldova）的国家顶级域名（ccTLD），本意也不是 `.markdown`

对于每个服务商所支持的域名，可以去看他们的文档:

- AWS: https://d32ze2gidvkk54.cloudfront.net/Amazon_Route_53_Domain_Registration_Pricing_20140731.pdf
- CF: https://dash.cloudflare.com/b273e7fb7d95f2ebd5830e5f7e79e821/registrar/register 并点击 "查看域注册文档和支持的 TLD"
