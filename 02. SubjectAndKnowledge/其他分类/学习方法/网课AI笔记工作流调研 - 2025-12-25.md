# 网课AI笔记工作流调研

## 提要

有段时间没有学习新知识了，在B站新添加了一个新的网课，打算每天看2集。
但我现在不想自己写笔记，我想要AI帮我把这个视频43集的笔记写出来。
不太确定当前的工作流。这这里闲聊一下。

即**重点在 ai 写视频笔记的工作流**，而非纯视频笔记工作流，不过后者在调研的过程中如果发现有比较有趣的也会在这里记录一下

## 字幕提取+gpt

我最早以前的做法是使用 “bilibili字母列表” 的浏览器扩展，他可以提取B站的字幕 (不能是无字幕/内嵌字母的情况)，而且有两种提取方式，一种是列表视图 (有时间信息)，一种是文章视图。

我通常可以选择性地从文章视图复制黏贴提取出来的字幕，再结构化地整理成笔记。看视频中途可能还会去再截截图，黏贴内容进去。
而如果有时间戳笔记的需求，应该是还能复制列表视图让gpt去做 (笔记可配合 Midia Extended 在 obsidian 中使用)

**优点**:

- 原信息与AI分离。
  这个扩展也自带了个ai总结的功能，但需要api。
  我想我直接复制所有字幕后黏贴到 gemini 中让他总结会更方便一些，也能更方便地添加提示词

**局限性**:

- 不太适用于无字幕/字幕内嵌而非B站字母的情况
  B站的自动字幕要up主开启
  (需要参考 [B站无字幕的处理方法](#B站无字幕的处理方法))
- 不太适合以画面为主的视频 (但这个目前好像都会有这个问题，似乎没有多媒体的视频阅读ai)。
  例如：视频中直接给你来一个思维导图，然后说：“这个主题分为三个的要点，如图，其中要点一扒拉扒拉”，但在讲的过程中又没有去提及这个要点叫什么名字。只通过语音进行的笔记是无法记录这种知识点的

## 一键总结工具

在开始之前，我还是搜了一下目前环境的情况。以前也有过一些评论区助手，刚搜了一些，有:

- BiliGPT
- NoteGPT
- ...等

**优点**:

其中 BiliGPT 还是开源的（官方在线版有次数限制，超出得用自己的api）。他们好像支持无字幕的情况。

**局限性**:

但 “一键总结” 我感觉的缺点是:

- 无参数/提示词可调。
  例如我有时并不想总结地太过精简，而有时又希望总结地更精简些，难以控制
- 不能切换模型，或使用非api的其他模型 (例如我买了模型，但没有买api)

## B站无字幕的处理方法

B站中，如果up主没有开启ai字幕，作为用户的我想自动获取视频中的说话内容，有什么好用的工具吗？（尽量不想下载视频文件）

deepseek v3.1:

对于B站视频，即使UP主没有开启AI字幕，也有工具可以帮你直接通过视频链接提取说话内容，**无需下载完整的视频文件**。

下面的表格整理了几种可行的方案，你可以根据自己的需求选择。

| 方案类型      | 工具/方法                | 核心优势                           | 是否免下载 | 备注/潜在不足                         |
| :-------- | :------------------- | :----------------------------- | :---- | :------------------------------ |
| 在线工具 (推荐) | 水印云                  | 直接粘贴链接，支持导出带时间轴的字幕，操作简单。       | 是     | 可能需要关注免费额度或次数限制。                |
|           | EasySub              | 纯网页端操作，支持150+语言识别和翻译。          | 是     | 对视频链接的可公开访问性有一定要求。              |
|           | WPS AI               | 与办公软件集成，识别后可一键生成摘要，适合整理学习笔记。   | 是     | 需要安装WPS客户端。                     |
| 系统功能      | iPhone 实时字幕 (iOS 18) | 系统级功能，完全离线处理，隐私保护好，可实时显示并复制字幕。 | 是     | 仅限苹果设备，且需系统为iOS 18或以上。          |
| 本地工具      | Bili2text            | 开源免费，功能专一，通过链接自动处理。            | 否     | 需要下载并安装Python环境，处理时会下载视频/音频文件。  |

| 其他方法      | 浏览器脚本/爬虫             | 自定义程度高。                        | 是     | 仅适用于视频已存在“AI小助手”字幕的情况，无法识别原始语音。 |

🎯 如何选择

根据你“尽量不想下载视频文件”的需求，可以优先考虑在线工具：
- 追求便捷：可以尝试 水印云 或 EasySub 这类在线网站，粘贴链接即可使用。
- 使用苹果设备：如果你的iPhone已升级到iOS 18，实时字幕功能是最直接、私密的解决方案。
- 需要整理笔记：WPS AI 的转写和总结功能可能更适合你。

⚠️ 注意事项

在使用这些工具时，有两点需要注意：
1.  尊重版权：提取的内容建议仅用于个人学习或笔记，尊重UP主的创作成果，避免违规传播或商用。
2.  注意限制：免费在线工具通常有次数、时长或导出限制。如果使用本地开源工具，则需要一定的技术学习成本。

如果需要了解某个特定工具更详细的操作步骤，我可以为你提供更具体的指引。

## 补充

### AI prompt

常用关键词，如:

:::note

以下是B站视频 https://www.bilibili.com/video/BV1YQ4y1A7MJ?p=2 的字幕列表

你需要帮我梳理成 Markdown 笔记

笔记偏好：分好章节、多用结构化表示、保留有用的细节而不要过度总结、不要过度加粗。
并在每个大章节的标题后的正文前适当地加入一个时间锚点以方便跳转 (只需要开始时间，不需要结束时间)，其中时间锚点的格式应为链接:
例如: `[12:37](https://www.bilibili.com/video/BV1YQ4y1A7MJ?p=6&t=757)`

------以下是字幕列表的内容------

:::

(不是固定的，你可以根据实际情况调整。如需要高概括高总结一点，还是要多保留一些细节)

其中，提供视频原链接方便 gpt 将视频链接与时间戳组合成可供跳转的锚点。
如果ai不会，时间戳这里你可以去教ai去弄标准的，也可以先让你以指定格式生成，后期再自己搞工具转换。

### B站视频链接如何进行时间跳转播放

就是url上加上 `t=秒书` 例如:

https://www.bilibili.com/video/BV1vL411u7bL?p=7&t=757

## 总结

目前我摸索的完整工作流是：

1. 浏览器扩展提取字幕 (或者像歌佬那样手动方案，用api拿到本地)
2. ai搭配提示词，梳理为markdown (分离提取和ai部分，高灵活度)
3. 开始正式看视频，看的过程可以对照笔记，和做一些手动编辑，增删优化
4. 看的过程中使用快捷键插入必要的截屏 (可参考 moy 佬的 quicker 动作，或用 videoNotes)
